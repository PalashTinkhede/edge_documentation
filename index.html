ZEUS Edge Platform Documentation

The ZEUS repository is a comprehensive edge computing infrastructure deployment toolkit designed specifically for Ubuntu 24.04 LTS systems. It serves as an automated installer and configuration system for creating a full-featured edge computing platform with monitoring, remote access, virtualization, and clustering capabilities.

1. Main Installation Script (install.sh)

Key Features

  • Root Privilege Enforcement: Ensures administrative access before execution.
  • Colorized Logging: Visual status updates for better readability.
  • Sequential Installation: Controlled rollout of components.
  • Forced System Reboot: Ensures all kernel modules and services initialize correctly.

Installation Sequence

  1. Performa: Monitoring Solution
  2. WeTTY: Remote Access Solution
  3. Wok: Virtualization Solution
  4. ClusterLabs: Cluster Management Solution

ZEUS Architecture Overview

  • Cluster-First Design: ZEUS is built around an autonomous edge cluster where all critical operations such as compute, high availability, monitoring, and virtualization run directly on the nodes.
  • Web Interface (React + Node.js): Provides a unified dashboard for administrators to view system status, manage resources, and initiate operations. This layer is only responsible for presentation and user interaction.
  • Central Server (Control Plane): Acts as an API gateway and coordination layer between the web interface and the cluster. It aggregates data, maintains metadata, and forwards commands, but does not run workloads or virtual machines.
  • Database: Connected exclusively to the central server to store historical metrics, cluster information, virtual machine metadata, and user/session data, keeping cluster nodes stateless from a management perspective.
  • Remote Access (Wetty): Runs on cluster nodes and provides secure, browser-based SSH access. The central server only exposes access points without handling shell execution.
  • Monitoring (Performa): Lightweight Performa satellite agents run on each node to collect CPU, memory, disk, and network metrics and send them to the central server for aggregation and visualization.
  • Virtual Machine Management (Wok & Kimchi): Enables web-based management of KVM virtual machines, storage pools, and networking. VM operations are executed directly on the cluster nodes hosting the workloads.
  • High Availability Clustering (Corosync & Pacemaker): Provides node membership, quorum, resource management, and automated failover entirely within the cluster, independent of the central server or web interface.
  • Cluster Execution Layer: Consists of multiple Ubuntu 24.04 LTS sub-nodes. Each node runs virtualization, monitoring, remote access, and clustering services, forming a self-healing and fault-tolerant environment.
  • Failure Isolation: If the web interface or central server becomes unavailable, running virtual machines and cluster services continue operating without interruption.

2. ClusterLabs Installation (Doc)


ClusterLabs is an open-source community that develops and maintains high-availability clustering software for Linux systems. It provides core components such as Pacemaker and Corosync to manage clustered resources, monitor node health, coordinate failover, and ensure uninterrupted service availability across distributed infrastructure environments.

Purpose

  • Node preparation for cluster membership
  • Installation of Pacemaker, Corosync, and PCS
  • Cluster service configuration

Key Operations

  1. System updates
  2. Time synchronization (chrony)
  3. Pacemaker stack installation
  4. PCSD daemon enablement
  5. Service state enforcement
  6. hacluster authentication setup

Installs the Pacemaker/Corosync stack to prepare nodes for High-Availability clustering.

3. Performa Package (Doc)

A Performa is a multi-server monitoring system with a web based front-end UI. It can monitor CPU, memory, disk, network, and of course your own custom metrics. Alerts can be configured to trigger on any expression, and send e-mails or fire web hooks. Timeline data can be stored on local disk or in Amazon S3.

Architecture Note: ARM is not supported for Performa. The binary is linux-x64 only.

  • Runs lightweight cron-based background process every minute.
  • Requires reliable outbound network to central server for monitoring.
  • Ensure Node.js runtime is properly installed prior to installation.
